1. Overview

Firefly is an agent observability platform. It gives developers real-time visibility into how their AI agents behave — LLM calls, tool invocations, errors, costs, and conversation trees.

Core value prop: Instrument your agent in under 5 minutes. See everything in under 5 seconds.

2. SDK Design

2.1 Python SDK — Core Patterns

# pip install firefly-sdk

from firefly import observe_agent, instrument_langchain

# Option A: One decorator for custom agents
@observe_agent("sales-outreach", workspace="acme")
def run(prompt: str):
    response = llm.invoke(prompt)
    return response

# Option B: Zero-code-change for LangChain
instrument_langchain()

2.2 TypeScript SDK

// npm install @firefly/sdk

import { observeAgent } from '@firefly/sdk';

@observeAgent('sales-outreach', { workspace: 'acme' })
async function run(prompt: string) {
  const response = await llm.chat(messages);
  return response;
}

// Auto-instrument LangChain.js
import { instrumentLangChain } from '@firefly/sdk/langchain';
instrumentLangChain();

2.3 Go SDK

// go get github.com/firefly/go-sdk

import "github.com/firefly/go-sdk"

tracer := firefly.NewTracer(firefly.Config{
  Workspace: "acme",
  APIKey:    os.Getenv("FIREFLY_API_KEY"),
})

ctx := tracer.Start(context.Background(), "sales-outreach")
defer tracer.End(ctx)

response, err := llm.Chat(ctx, messages)

2.4 Shared SDK Contract

  • Batch flush: every 1 second or 100 events
  • Offline queue: disk-backed, syncs on reconnect
  • Idempotent: dedup by trace_id
  • Test mode: captures traces in-memory for unit tests
  • Single ingestion endpoint for all languages

POST /api/v1/ingest/traces
Authorization: Bearer {api_key}
Content-Type: application/json

[
  {
    "trace_id": "abc-123",
    "agent_id": "sales-outreach",
    "workspace": "acme",
    "timestamp": "2026-05-10T14:23:01Z",
    "event_type": "llm_call",
    "llm_provider": "openai",

    "prompt_tokens": 342,
    "completion_tokens": 128,
    "latency_ms": 840,
    "status": "success",
    "parent_span_id": "abc-123.parent"
  }
]
  • Response: 202 Accepted (async processing)
  • Schema validation via Pydantic/Zod
  • Write to ClickHouse time-series table
  • Fan out via WebSocket to connected clients

6. Frontend

6.1 Key Screens

  • Dashboard — Per-agent health cards (success rate, cost/hr, error rate, active conversations)
  • Trace Explorer — Search/filter traces by agent, time range, error type. Expand conversation tree.
  • Conversation Replay — Step-by-step visualization: each LLM call, tool call, and response in order.
  • Alerts — Configure alert rules. View alert history.
  • Cost Analytics — Per-agent cost over time, per-model breakdown.

6.2 Real-time

WebSocket server pushes new traces to frontend as they arrive. No polling. Dashboard updates live.

7. Alert Engine

POST /api/v1/alerts
{
  "agent_id": "sales-outreach",
  "metric": "success_rate",
  "threshold": 0.90,
  "window_minutes": 60,
  "channel": "slack",
  "webhook_url": "https://hooks.slack.com/..."
}

Supported metrics: success_rate, error_rate, avg_latency_ms, cost_per_hour, token_spike

Supported channels: Slack, email, webhook. Rules stored in PostgreSQL, evaluated every 30 seconds via cron.

8. Onboarding Flow

  1. User signs up → gets workspace + API key
  2. pip install firefly-sdk (or npm / go get)
  3. Add one line to agent code: instrument_langchain() or @observe_agent("my-agent")
  4. Run agent → traces appear on dashboard in real-time
  5. Done. No config files. No YAML. No hell.
